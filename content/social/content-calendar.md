### Day 20 (Fri) — Engagement: Behind the Build
**Twitter:**
We built CueDeck as a single-file HTML app.

No framework. No build system. Pure HTML/CSS/JS + Supabase.

Why? Because production teams need something that loads instantly, works offline-first, and never shows a loading spinner during a live event.

Sometimes the best architecture is the simplest one.

**LinkedIn:**
A technical decision we made early on: CueDeck's production console is a single HTML file.

No React. No Next.js. No build step. Just HTML, CSS, and JavaScript loaded directly in the browser.

Why?

1. Zero load time — the console opens instantly
2. Offline resilience — the UI never breaks if the connection drops
3. Deployment simplicity — one file, one URL, every operator
4. No framework churn — it works today and will work in 5 years

The backend is Supabase (Postgres + real-time subscriptions + edge functions). The frontend is intentionally simple because when you're running a live event, reliability beats novelty every time.
